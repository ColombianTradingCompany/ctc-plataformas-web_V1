Clientes — las 4 superficies que la gente toca
🏠

CTC Home

Público

Marketing corporativo + captura de leads. El "Escríbenos" y los servicios de "Más allá de la exportación" (CTC Tech, Co-Create, Varietales) ya no arman un mailto:: crean un lead con una cuenta de plataforma (Google, o creada en su nombre) y disparan un correo de bienvenida automático.

Ya no es "sin backend": una Server Action pública (service-role) inserta el lead en leads y crea/vincula la cuenta — Co-Create → comprador Cherry Picked, el resto → productor Kaffetal Regal. El pipeline se tría en /bcp/leads.

🌱

Kaffetal Regal

Productor

Portal del caficultor: login real (email+contraseña o Google), CRUD de fincas/lotes con expediente EUDR completo, y la Ficha Técnica dividida en 4 sub-etapas secuenciales (FT → FT2 → EUDR → Video). El dashboard es ahora un HUB de módulos (tarjetas grandes con datos clave: Información general, Mis fincas, Mis lotes, Envío de muestras, Retroalimentación, Certificación, Contratos, Más allá de la exportación) — se abre uno a la vez. La superficie más activa del proyecto.

Dos novedades clave: el productor puede solicitar los servicios "Más allá de la exportación" (CTC Tech / Varietales) desde su propio panel, y su hilo de "Retroalimentación y ayuda" es una conversación bidireccional real con CTC — incluye los espejos de sus solicitudes de leads.

☕

Cherry Picked

Comprador

Vitrina para tostadoras europeas: catálogo alimentado solo por publicaciones de BCP, carrito, checkout atómico, niveles de membresía, sample packs.

La subasta Tyrian (TyrianSection.tsx) sigue siendo 100% demo estática — deliberadamente aplazada, aún no empezada.

🛡️

BCP · Business Control Panel

Staff CTC

La única superficie que usa el equipo de CTC: login 2FA; un Panel de triaje (KPIs gráficos + feed de tareas Done/TBD); aprobación de fincas; tablero Kanban de intake (FT/FT2/EUDR/Video/Muestra); Arena; contratos; publicación al catálogo; /bcp/productores; y /bcp/leads — ya no un placeholder: un tablero kanban por servicio (Escríbenos / CTC Tech / Co-Create / Varietales) para responder los leads de CTC Home por correo.

Cada edición EUDR que BCP hace a nombre de un productor se auto-registra en el log de comunicación. El Panel triage y el estado Done/TBD viven en la tabla bcp_task_state, compartida entre los dos admins.

Código

Un solo repo, cuatro caras. App Router + TypeScript + React 19, Turbopack. CSS Modules a mano en las 3 superficies públicas; shadcn/ui + Tailwind solo dentro de BCP.

Enrutador

Identidad y acceso a datos — cómo cada plataforma habla con Supabase
🔑

Supabase Auth

Identidad

Dos esquemas conviven: productores/compradores usan email+contraseña o Google OAuth normales; BCP envuelve eso con su propio password+OTP de 2 factores.

🔌

Cliente Supabase (navegador)

Acceso directo

Kaffetal Regal y Cherry Picked llaman a supabase-js directo desde el navegador — no existe una API intermedia para ellos. La seguridad vive entera en RLS + guardas.

📜

Server Actions (BCP)

Contrato admin

BCP nunca toca Supabase desde el navegador. Cada mutación es una Server Action que corre requireAdmin() por su cuenta, más 2 route handlers dedicados (2FA, asesor IA).

Lógica de negocio — dónde viven las reglas reales
🧱

RLS + guardas "ironclad"

Reglas

Dos capas: RLS decide de quién son las filas visibles; un trigger BEFORE UPDATE decide además qué columnas/transiciones son legales, y las políticas cubren también DELETE. Se sumaron dos guardas nuevas: guard_finca_protected_columns (nadie que no sea CTC cambia status/eudr_cert_shared, y las declaraciones EUDR quedan congeladas una vez aprobada la finca) y guard_lot_protected_columns (protege el nivel de riesgo, la efectividad de mitigación y el responsable — determinaciones de CTC).

Las llamadas service-role tienen auth.uid() nulo y las guardas las dejan pasar por diseño. Hallazgo de esta ronda: fincas no tenía NINGUNA guarda de columnas — un productor podía (vía API directa) auto-aprobarse o auto-compartirse la certificación. Verificado con SQL simulando la sesión del productor: auto-aprobación y auto-share BLOQUEADOS, edición legítima PASA.

⚙️

Funciones SECURITY DEFINER1 de 5 intencional

Lógica atómica

place_order() es la única invocada a propósito (desde el navegador del comprador). Las otras 4 son triggers automáticos que Postgres expone sin querer vía RPC.

🤝

Servicios externos

Rentado

Resend entrega el OTP de BCP y ahora todos los correos de leads (bienvenida + respuestas de CTC), desde info@ctcexport.com con el dominio verificado (env EMAIL_FROM). La API de Anthropic resuelve el asesor "¿Y ahora qué?" — su ruta/lib/columnas siguen, pero la UI está desactivada (dormante).

Archivos
