## [2026-07-12] Finca location rendering + comm-log internal links
- Finca map: new shared mapPreviewUrl() (src/lib/eudr.ts) -- a Google Static
  Maps <img>, no WebGL -- now renders a saved pin/polygon outside the edit
  modal on both sides: a new thumbnail on the producer's own finca card
  (AppDashboard.tsx, Kaffetal Regal) and BCP's existing FincaEudrEditor.tsx
  map preview (refactored off its own local duplicate onto the shared
  helper). Not a new scenario -- refines the existing "producer sets
  finca geolocation" flow's node behavior (Kaffetal Regal finca card,
  BCP finca review card) rather than adding a new one.
- Comm-log internal links: producer_comm_log gained real finca_id/lot_id FK
  columns (previously only a free-text context_label). logProducerComm()
  and every BCP call site (fincas/lotes/productores pages, the EUDR
  auto-log in bcp/actions.ts) now pass the real id. The producer's
  "Retroalimentación y ayuda" panel (AppDashboard.tsx) renders each note's
  context label as a clickable link that opens the referenced finca modal
  or lot ficha directly -- refines the existing BCP<->producer
  communication node/flow, not a new one.
- Bug fix: approveFinca() (bcp/actions.ts) was gating on a dead legacy
  column (eudr_polygon, PostGIS) instead of eudr_polygon_geojson, silently
  blocking approval of any >4ha finca even after the producer drew a
  polygon. Worth a DICT-level note if the EUDR approval flow's description
  ever mentions the polygon gate mechanics.
- Bug fix: comm-log matching was by free-text label, which could leak
  notes across two different producers whose finca/lot shared a name --
  fixed as part of the FK change above.
- Bug fix: nested <button> inside <button> hydration error in
  FincaMapPicker.tsx (FieldInfo's info icon next to "Ingresar puntos
  manualmente") -- no doc-visible behavior change, purely a correctness fix.
(commit: 8eb44fa)

## [2026-07-12] Static map: terrain, not hybrid (EEA restriction) + Ubicación polygon count
- Follow-up to the mapPreviewUrl() entry above: Google's Static Maps API
  returns 403 for maptype=hybrid/satellite on EEA-billed accounts (CTC's
  Google Cloud billing is in Germany), so the static previews now use
  maptype=terrain. The interactive picker (Maps JavaScript API) is exempt
  and stays on hybrid. Worth a note in whatever DICT entry describes the
  Google Maps integration: the two map surfaces intentionally use
  different imagery for a billing-region reason, not by accident. Also
  required two one-time Google Cloud console steps (enable Maps Static
  API on the project + add it to the key's API-restrictions allowlist).
- BCP's read-only "Ubicación" line now says "Polígono de N vértices" for
  polygon-only fincas instead of the misleading "no capturada".
(commits: c8d3b16, 200e0df)
