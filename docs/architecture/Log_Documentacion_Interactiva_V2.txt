## [2026-07-12] Static map: terrain, not hybrid (EEA restriction) + Ubicación polygon count
- Follow-up to the mapPreviewUrl() entry above: Google's Static Maps API
  returns 403 for maptype=hybrid/satellite on EEA-billed accounts (CTC's
  Google Cloud billing is in Germany), so the static previews now use
  maptype=terrain. The interactive picker (Maps JavaScript API) is exempt
  and stays on hybrid. Worth a note in whatever DICT entry describes the
  Google Maps integration: the two map surfaces intentionally use
  different imagery for a billing-region reason, not by accident. Also
  required two one-time Google Cloud console steps (enable Maps Static
  API on the project + add it to the key's API-restrictions allowlist).
- BCP's read-only "Ubicación" line now says "Polígono de N vértices" for
  polygon-only fincas instead of the misleading "no capturada".
(commits: c8d3b16, 200e0df)

## [2026-07-12] Producer mobile layout fix + finca delete/revision lifecycle
- Mobile: the producer dashboard's "Mis fincas" strip and "Mis lotes" rows
  overflowed the viewport in portrait (only usable in landscape). Root
  cause was grid items defaulting to min-width:auto; fixed with min-width:0
  on .acard/.lotrow, kanban stepper now wraps, long CTC_ ref wraps. Pure
  CSS/layout, no doc-model change.
- Finca deletion lifecycle (worth a DICT note + possibly a small scenario
  on the finca node): a producer can now DELETE a finca while CTC hasn't
  accepted it (status != approved) AND no lot of it has reached the Arena
  (stage < fila_arena) -- deleting cascades its still-pending lots (with a
  warning naming them). New RLS policy fincas_delete_own_not_committed
  replaces fincas_delete_own_before_eudr. Once the finca is approved or has
  a committed lot, the "Eliminar" button is replaced by "Solicitar revisión
  de datos", which opens a prefilled email to info@ctcexport.com; a full
  deletion affecting committed lots is handled by CTC on that thread. New
  shared helpers fincaSelfDeletable()/pendingLotsOfFinca()/isLotCommitted()
  in data.ts keep the button state and the RLS policy in sync. Lot type
  gained fincaId (real FK) for robust finca<->lot matching (the old
  name-based match could confuse two same-named fincas).

## [2026-07-13] Producer answer vs CTC-evaluated split (Respuesta de Productor)
- New fincas.eudr_producer_answers jsonb snapshots the producer's five
  declarations + geolocation (deforestationFree, legalProduction, tenure,
  plantingDate, productionSystem, lat/lng/polygon). The existing eudr_*
  columns now hold CTC's EVALUATED value -- that's what the status/chip,

  now have two layers (producer's declaration vs CTC's evaluation).
- Producer FincaModal edits their own answer (seeds from eudr_producer_answers,
  fallback to eudr_* for legacy fincas). saveFinca snapshots the answer and
  lets eudr_* follow the producer ONLY on fields the producer actually changed
  -- preserving a prior CTC override on untouched fields. When CTC's value
  differs, the producer sees an amber banner ("CTC ajustó… → X; su respuesta
  original se conserva") -- verified live.
- BCP FincaEudrEditor: each of the five declarations shows "Respuesta de
  Productor: X" with a one-click "Confirmar" (adopt) or a normal edit
  (override). Whatever is saved to eudr_* is CTC's value; the certificate
  prints only that, never the producer snapshot.
