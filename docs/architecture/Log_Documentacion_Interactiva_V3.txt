## [2026-07-14] Kaffetal Club: códigos de miembro que habilitan contratos + catálogo
- New feature spanning DB + KR + BCP. DB: new service-role-only table
  `club_member_codes` (code KC-XXXX-XXXX, note, created_by, redeemed_by/at,
  revoked_at; profile FKs ON DELETE SET NULL) + new guard-protected column
  `producer_profiles.club_member_since` + new guard trigger
  `guard_producer_protected_columns` (INSERT+UPDATE -- first guard that also
  covers INSERT, because producer_profiles_insert_own allows self-insert).
- Producer flow (Kaffetal Regal): "Mis contratos" hub module is now
  member-only. Non-members see a locked tile + an explanation card with a
  code-redemption input (AppDashboard.tsx); redeeming calls the new server
  action src/lib/club/actions.ts redeemClubCode() -> claims the code
  (concurrency-safe conditional update), upserts club_member_since, writes
  audit_log ('club_membership'/'code_redeemed') + a welcome note into
  producer_comm_log (shows in Retroalimentación y ayuda).
- BCP flow: new page /bcp/club (emit code with note, revoke unredeemed codes,
  member list with "Retirar del Club" -> revokeClubMembership) via
  clubActions.ts; new sidebar link; /bcp/productores shows "Kaffetal Club ✓"
  chip per member.
- Enforcement (business-logic gates): contractActions.signContract refuses if
  the lot's producer has no club_member_since; catalogActions.publishLot same
  check (defense in depth). Contracts are still auto-created at grading for
  everyone -- membership gates SIGNING, so late joiners find their pending
  contract waiting.
- Doc impact when wrapping: likely a new SCENARIO (producer redeems a member
  code -> membership -> contract signing unblocked), a DICT entry for
  "Kaffetal Club" cross-linked from the contracts/catalog nodes, a node-card
  note update on the BCP contracts + catalog nodes (new gate), and possibly a
  new node for /bcp/club under Business logic.
(commit: a8abc78)

## [2026-07-14] Kaffetal Club v2: modelo "Pasaporte" (narrativa + pipeline + campañas)
- Narrative reframe: membership = "Pasaporte del Kaffetal Club", code =
  "Número de Pasaporte". Producer gate in "Mis contratos" rewritten as an
  exporters-club passport (request + grant), incl. a "Solicitar mi Pasaporte"
  button -> deduplicated producer-authored producer_comm_log note (service
  role; the RLS insert policy wouldn't allow a standalone producer note).
- club_member_codes gained: kind ('estandar' KC- / 'campana' KCX- secret
  prefix), campaign label, assigned_to/assigned_at, email_sent_at/email_error
  (migration club_passport_pipeline). Assigned codes are personal: redemption
  by anyone else is refused ("emitido a nombre de otro productor").
- BCP /bcp/club rebuilt as a 3-column passport kanban: Elegibles (>=1 lot past
  Muestras, grayed until a galardón; "Emitir Pasaporte" is ALSO server-gated on
  a galardonado lot in emitPassportForProducer), Pendiente de confirmación
  (assigned+sent, email status with Reenviar + Revocar), Miembros activos
  (Retirar del Club). Below: campaign-passport creation ("Fundadores", ...;
  assigned to anyone bypassing the Arena gate, or unassigned to hand out) and
  a full passport ledger with the 3 states Emitido/Pendiente/Activo.
- Assigned emission delivers the número twice: passport email
  (src/lib/email/clubEmails.ts via the shared Resend sender now exported from
  leadEmails.ts, failures persisted on the code row leads-style) + a
  producer_comm_log note in Retroalimentación y ayuda.
- Doc impact when wrapping: update the Kaffetal Club DICT entry + /bcp/club
  node to the passport/kanban model; the redemption SCENARIO gains the
  request->emit->confirm pipeline and the campaign bypass variant; new email
  wire BCP -> Resend for passport delivery.
(commit: 2ac404e)

## [2026-07-14] Campañas de Pasaportes como entidad propia + reordenado /bcp/club
- New table club_campaigns (name unique, created_by, created_at;
  service-role-only) -- campaign passports now carry campaign_id FK
  (ON DELETE SET NULL); the loose `campaign` text column on
  club_member_codes was dropped (migration club_campaigns_entity, with
  backfill). Campaign names resolve via the FK join everywhere (redemption
  welcome note/audit, BCP cards, resend email).
- /bcp/club reordered: Campañas block at the TOP (create form + one clickable
  card per campaign with activos/pendientes/sin-asignar counts) -> passport
  kanban below -> ledger. New detail page /bcp/club/campanas/[id] manages one
  campaign: emit passports rooted under it (assigned to any producer -- Arena
  bypass -- or a BATCH of up to 50 unassigned hand-out codes) and
  resend/revoke its existing ones.
- clubActions: createCampaign + emitCampaignPassports(campaignId, formData)
  replace the old createCampaignPassport(label) action.
- Doc impact when wrapping: the Kaffetal Club DICT entry + /bcp/club node gain
  the campaign-entity structure and the campanas/[id] subpage.
(commit: 83b078f)
